## ğŸ¨ Brand Voice & Tone

### Voice Attributes
- **Knowledgeable**: Expert-level insights without being condescending
- **Approachable**: Friendly and conversational, not academic
- **Practical**: Focus on actionable advice over theory
- **Encouraging**: Supportive and motivating, celebrating wins
- **Authentic**: Real experiences and honest perspectives

### Tone Variations by Content Type
- **Educational Content**: Clear, patient, step-by-step
- **Marketing Copy**: Energetic, benefit-focused, inspiring
- **Community Posts**: Casual, supportive, celebratory
- **Technical Docs**: Precise, thorough, example-rich

### Language Guidelines
- Use "you" and "we" for connection
- Avoid jargon without explanation
- Include real-world examples
- Balance technical accuracy with accessibility
- Use active voice predominantly

## âœï¸ Writing Standards

### Headlines & Titles
- **Courses**: Clear outcome + benefit (e.g., "Master Cursor Rules for 70% Faster Development")
- **Products**: Value proposition focused (e.g., "200+ Battle-Tested Cursor Rules")
- **Blogs**: SEO-optimized with keywords (e.g., "How to Use Cursor AI: Complete 2025 Guide")
- **Emails**: Curiosity or benefit driven (e.g., "The 10-minute trick that changed my coding")

### Content Structure
1. **Hook**: Grab attention in first 2 sentences
2. **Promise**: Clear value proposition
3. **Roadmap**: Preview what's covered
4. **Main Content**: Logical flow with subheadings
5. **Examples**: Real-world applications
6. **Summary**: Key takeaways
7. **CTA**: Clear next action

### Formatting Standards
- Use H2 (##) for main sections
- Use H3 (###) for subsections
- Bold for emphasis, not italics
- Bullet points for lists > 3 items
- Code blocks with language specification
- Include visuals every 300-500 words

## ğŸ“Š Content Types & Requirements

### Course Content
- **Module Length**: 4-6 lessons per module
- **Lesson Length**: 15-20 minutes of content
- **Practice Ratio**: 40% theory, 60% hands-on
- **Resources**: Downloadable summary for each module
- **Assessment**: Quiz or project per module

### Product Content
- **Description Length**: 500-1000 words
- **Benefits**: Minimum 5 clear benefits
- **Features**: Technical specifications
- **Social Proof**: 3+ testimonials
- **Visuals**: Screenshots/demos required

### Blog Posts
- **Length**: 1000-2500 words typically
- **SEO**: Focus keyword in title, URL, H2s
- **Images**: Featured image + 2-3 in-post
- **Links**: 3-5 internal, 2-3 external
- **CTA**: Email capture or product

### Email Content
- **Subject Length**: 30-50 characters
- **Preview Text**: 40-100 characters
- **Body Length**: 150-300 words
- **CTA**: Single, clear action
- **Mobile**: 60% will read on mobile

## ğŸ”„ Content Workflow

### Creation Process
1. **Ideation**: Use idea files in each content folder
2. **Research**: Validate demand and competition
3. **Outline**: Create structure before writing
4. **Draft**: Write without editing
5. **Edit**: Content, clarity, and brand alignment
6. **Review**: Technical accuracy and quality
7. **Optimize**: SEO and conversion elements
8. **Publish**: Follow platform-specific process
9. **Promote**: Execute promotion plan
10. **Analyze**: Track performance metrics

### Review Cycles
- **Quarterly**: Full content audit
- **Monthly**: Performance review
- **Weekly**: New content planning
- **Daily**: Community engagement
